## 緣起

最近跟同事分享怎麼用 Claude Code 和 Cowork，過程中我試著用最白話的方式解釋 AI Agent 到底在幹嘛。講完之後覺得這些概念值得記下來，因為理解這些「本質」之後，你對 AI 工具的掌握度會完全不同。

## 關鍵收穫

### 一、Agent 就是實習生，差別在聰明程度

我最常用的比喻：AI Agent 就是一個實習生。

你今天找一個聰明的實習生，跟他說「把這些音檔全部轉成文字，然後整理出幾個重點方向給我」——他就去做了。笨一點的實習生，可能要自己去找音轉文字的工具，一步一步搞半天；但聰明的實習生（像 Claude Code），他自己就全部搞定了，工具自己找、流程自己跑。

這就是現在 Agent 跟以前聊天機器人最大的不同：**你不用告訴他「怎麼做」，你只要告訴他「做什麼」。**

### 二、CLAUDE.md 就是這個實習生的 JD

每個 AI Agent 進到一個工作空間（資料夾），第一件事就是去看一個叫 `CLAUDE.md` 的檔案。這個檔案就像是實習生的 **Job Description（JD）**——告訴他：你在這個專案裡該做什麼、有什麼規矩要遵守。

最基本的可能就寫一行：「你一定要用繁體中文跟我講話。」進階一點，就會寫清楚工作要求、品質標準、資料在哪裡。

重點是：**一個資料夾就是一個工作空間，每個工作空間可以有自己的 CLAUDE.md。** 這個概念跟寫程式一模一樣——程式專案一定會有一份文件說明這個專案在幹嘛、資料夾結構長怎樣。AI Agent 也是。

最上層的 CLAUDE.md 寫大方向，例如「這個資料夾裡有 NPO 相關文件、有音檔資料」。每個子資料夾裡面可以再有自己的說明文件，更細地描述裡面有什麼。

Agent 工作的時候就是這樣運作的：先看最上層的說明 → 根據你給的任務判斷要去哪裡找資料 → 像查字典一樣，一層一層找到他需要的東西。

所以你不需要什麼都自己整理好再餵給他——**你只要把「地圖」畫好，他會自己去探索。**

### 四、Skill 就是直接植入大腦的技能晶片

這是我覺得最精彩的概念。Skill 其實就是以前大家在玩的 GPTs，但更結構化。一個 Skill 包含三個東西：

1. **角色定義（System Prompt）**：這個技能是什麼、標準是什麼
2. **知識資料（Data/References）**：輔佐這個技能需要的參考資料
3. **程式碼（Code）**：可以直接執行的腳本，Agent 不需要讀懂，直接跑

以前實習生某個技能很差，你要送他去上課。現在？**直接把 Skill 塞進去，他瞬間就會了。** 就像植入一個晶片，馬上獲得新能力。

而且 Skill 存在的一個重要原因是**省 Token**。如果什麼都寫在主文件裡，Agent 每次都要讀一大堆東西；有了 Skill，他像查字典一樣，需要什麼技能才去載入，不需要的就不看。

### 五、Agent 自己會帶團隊

這是讓我自己都覺得驚訝的部分。Claude Code 在處理複雜任務的時候，會自己拆分成多個 Sub-agent 去工作。

我拿自己寫網站的經驗來說：我只跟他講「我要寫一個功能」，他會：
1. 先叫一個 **TPM（技術專案經理）** 來寫規劃書
2. 規劃書我確認 OK 之後，他叫一個**後端工程師**去寫程式
3. 寫完之後，再叫一個 **Senior Engineer** 來做 Code Review
4. Review 有問題就改，改完才回報給我

以前這要請三個人，現在 Agent 自己就是一個團隊。所以我今年第一季，一行 Code 都沒寫——全部都是用這種方式完成的。

### 六、最後活下來的是 PM，不是工程師

聊到這裡，大家可能會問：那工程師怎麼辦？

坦白說，**只會寫 Code 的工程師，生存空間會越來越小。** 因為 Agent 已經能處理大部分的開發工作。但真正重要的兩個角色反而浮上來了：

- **PM（產品經理）**：能定義問題、知道「為什麼要做」的人
- **Architect（架構師）**：能判斷 Agent 產出的東西好不好、架構對不對的人

未來的團隊可能就是兩個人——一個 PM、一個 Architect——就能做以前一整個團隊的事。問題定義跟驗證（Validation），才是這個時代最值錢的能力。

## 我的看法

我自己今年一行 Code 都沒寫，但老實說，疲累程度跟去年差不多。原因是什麼？**認知債。** 你省了寫的過程，但判斷的過程一點都沒省。你要花大量時間看 Agent 給你的東西、判斷好或不好、思路還是要自己走過一遍。

所以 AI 並沒有讓工作變輕鬆——它改變的是工作的「形狀」。從動手做，變成動腦判斷。

對基金會的同仁來說，我覺得最重要的起步就是：**把你的工作想清楚、結構化地描述出來。** 這件事做好了，不管 AI 工具怎麼演進，你都能用。
