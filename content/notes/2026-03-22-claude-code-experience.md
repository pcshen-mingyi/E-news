## 初次接觸 Claude Code

作為一個每天都在寫程式的開發者，Claude Code 完全改變了我的工作流程。它不只是一個程式碼補全工具，更像是一位隨時待命的 pair programming 夥伴。

## 最喜歡的功能

### 1. 上下文理解能力

Claude Code 能夠理解整個專案的結構，不只是單一檔案。當你問它關於某個 bug 的問題時，它會自動去讀取相關的檔案來理解上下文。

### 2. 多檔案同時編輯

需要重構一個函式名稱？Claude Code 可以同時找到所有引用的地方並一次修改。

### 3. 測試生成

寫完功能後，只要說一句「幫我寫測試」，它就能根據程式碼邏輯產生合理的測試案例。

## 使用建議

- 給予清楚的上下文和需求描述
- 善用 CLAUDE.md 檔案記錄專案規範
- 不要盲目接受所有建議，保持 code review 的習慣
- 複雜任務先讓它規劃，確認方向後再動手
